| 原则 | 说明 |

## 2. 技术选型

### 2.1 选型总览

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|----------|
| **VitePress** | ^1.6 | 静态站点生成 | 基于 Vite 构建速度快、Markdown + Vue SFC 原生支持、内置全文搜索 |
| **Vue 3** | ^3.4 | 组件系统 | Composition API 简洁、VitePress 内置无需额外安装 |
| **TypeScript** | ~5 | 类型安全 | VitePress 配置原生支持、提高可维护性 |
| **iframe Sandbox** | — | 样式隔离 | 比 Shadow DOM 隔离更彻底、兼容性更好、支持 JS 交互 |
| **Netlify** | — | 部署平台 | 零配置 CI/CD、自动 HTTPS、自定义域名、全球 CDN |

### 2.2 技术选型对比

#### iframe vs Shadow DOM

| 维度 | iframe (选中) | Shadow DOM |
|------|--------------|------------|
| 样式隔离 | **完全隔离**，绝对不泄漏 | 部分隔离，部分全局 CSS 仍穿透 |
| JS 隔离 | **完全隔离**，独立全局对象 | 不隔离，共享主线程全局对象 |
| 交互支持 | 完全支持（点击、拖拽等） | 完全支持 |
| 安全性 | **高**，sandbox 可限制权限 | 无安全边界 |
| 渲染开销 | 略高（独立的文档上下文） | 较低 |
| 调试难度 | 较复杂（独立 DevTools 上下文） | 较简单 |

> **结论**：对于 CSS 效果展示场景，效果代码通常是用户编写的不可信内容，iframe 的隔离安全性远重要于微小的性能差异。

#### VitePress vs 其他方案

| 维度 | VitePress (选中) | Docusaurus | 自建 Vue 项目 |
|------|-----------------|------------|--------------|
| 上手成本 | **低**，最小配置即可运行 | 中，需要较多配置 | 高，需从零搭建 |
| Markdown 支持 | **原生**，支持 Vue SFC 嵌入 | 原生，支持 React 组件 | 需自行集成 |
| 搜索 | **内置** local search | 内置 | 需集成第三方 |
| 主题定制 | Vue 组件覆盖，灵活 | React 组件覆盖 | 完全自定义 |
| 构建速度 | **快**（基于 Vite） | 中（基于 Webpack） | — |

### 3.2 模块职责

#### 内容管理模块

负责所有效果内容的存储、组织和元数据管理。

- 每个效果对应一个 `.md` 文件
- 按分类存放在 `docs/effects/` 下的子目录中
- 通过 YAML Frontmatter 管理标题、描述、标签、日期
- 效果代码通过 `<script setup>` 的 JavaScript 模板字符串管理

#### 主题与组件模块

负责页面的渲染呈现和交互功能。

- 扩展 VitePress 默认主题
- 提供 `CssDemo` 全局 Vue 组件
- 实现 iframe 预览、代码展示、复制等交互
- 适配暗色/亮色主题

#### 构建与部署模块

负责将源代码编译为静态站点并部署上线。

- VitePress 标准构建流程
- 构建时扫描文件系统生成侧边栏配置
- Netlify 自动部署配置

## 7. 核心组件 CssDemo 详细设计

### 7.1 组件接口

```typescript
interface CssDemoProps {
  /** HTML 结构代码 */
  html?: string      // 默认: ''
  /** CSS 样式代码 */
  css?: string       // 默认: ''
  /** JavaScript 交互代码 */
  js?: string        // 默认: ''
  /** 预览区域高度（CSS 单位） */
  height?: string    // 默认: '300px'
  /** 预览区域背景色 */
  bg?: string        // 默认: '#f7f7f7'
  /** 是否禁用沙箱脚本执行 */
  noScript?: boolean // 默认: false
}
```

### 7.2 组件状态

```typescript
interface CssDemoState {
  showCode: boolean   // 是否显示源代码区域
  loaded: boolean     // iframe 是否加载完成
  copied: boolean     // 是否已复制代码（用于反馈提示）
  iframeRef: Ref      // iframe DOM 引用
}
```

### 7.3 iframe 渲染机制

#### 渲染流程

```
1. 组件接收到 html/css/js props
       │
2. 计算 iframeDoc（响应式 computed）
       │
       ├── 组装基础 HTML 骨架（DOCTYPE, html, head, body）
       │
       ├── 注入内置样式（reset, flex居中, 字体）
       │     └── + 拼接 props.css
       │
       ├── 注入 HTML 内容：props.html
       │
       └── 条件注入 JavaScript：props.js
             └── 注意：<\/script> 转义防止闭合宿主 script 标签
       │
3. 通过 :srcdoc="iframeDoc" 绑定到 iframe
       │
4. iframe 加载 → @load 事件 → loaded = true → 隐藏加载动画
```

#### srcdoc 输出示例

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <style>
    /* 内置 reset + flex 居中 */
    * { margin: 0; padding: 0; box-sizing: border-box; }
    body {
      display: flex;
      justify-content: center;
      align-items: center;
      min-height: 100vh;
      padding: 20px;
      font-family: system-ui, -apple-system, sans-serif;
    }
    /* 用户 CSS */
    .my-btn { color: red; }
  </style>
</head>
<body style="background:#f7f7f7">
  <!-- 用户 HTML -->
  <button class="my-btn">Hello</button>

  <!-- 用户 JavaScript -->
  <script>console.log('hello')</script>
</body>
</html>
```

### 7.4 沙箱安全策略

| sandbox 值 | 说明 |
|-----------|------|
| `allow-scripts` | **开启** — 允许执行脚本，支持交互效果 |
| `allow-same-origin` | **关闭** — 禁止访问宿主页面 DOM/Cookies |
| `allow-top-navigation` | **关闭** — 禁止 iframe 修改顶层页面 URL |
| `allow-forms` | **关闭** — 禁止表单提交 |
| `allow-popups` | **关闭** — 禁止弹出窗口 |

### 7.6 交互功能实现

#### 展开/收起源码

```typescript
const showCode = ref(false)
// 点击 "查看代码" / "收起代码" 按钮切换
@click="showCode = !showCode"
```

#### 一键复制

```typescript
function copyCode() {
  const text = [
    props.html ? `<!-- HTML -->\n${props.html}` : '',
    props.css ? `/* CSS */\n${props.css}` : '',
    props.js ? `// JavaScript\n${props.js}` : '',
  ].filter(Boolean).join('\n\n')

  navigator.clipboard.writeText(text).then(() => {
    copied.value = true
    setTimeout(() => { copied.value = false }, 2000)
  })
}
```

#### 新窗口打开

```typescript
function openNewWindow() {
  const win = window.open('', '_blank')
  if (win) {
    win.document.write(iframeDoc.value)
    win.document.close()
  }
}
```

### 8.3 目录到侧边栏的映射规则

|----------|--------|

### 8.4 标题提取策略

```typescript
function extractTitle(filePath: string): string | null {
  const content = fs.readFileSync(filePath, 'utf-8')
  // 从 Frontmatter 提取 title 字段
  const match = content.match(/^title:\s*(.+)$/m)
  // 匹配格式：title: 效果名称
  return match ? match[1].trim() : null
}
```

若 Frontmatter 中没有 `title` 字段，则使用文件名（`-` 替换为空格）作为后备标题。

## 11. 扩展设计

### 11.1 添加新效果

**操作步骤**：

```
1. 进入对应分类目录（如 effects/buttons/）
2. 创建新文件，如 pulsate-btn.md
3. 编写 Frontmatter 和内容
4. 提交代码
```

**自动生效机制**：

```
文件创建 → 下次构建 →
  buildEffectsSidebar() 扫描目录 →
  读取 pulsate-btn.md title →
  侧边栏自动追加「脉冲按钮」条目 →
  路由自动生成 /effects/buttons/pulsate-btn/
```

### 11.2 添加新分类

**操作步骤**：

```
1. 在 effects/ 下创建新目录，如 effects/inputs/
2. 创建 inputs/index.md 作为分类概览页
3. 创建具体效果文件
4. 提交代码
```

**自动生效机制**：

```
新目录 → 下次构建 →
  buildEffectsSidebar() 发现新子目录 →
  新增「Inputs」分类 →
  目录下的 .md 文件自动成为子条目
```

### 11.3 添加新全局组件（进阶）

如果需要扩展更多功能组件（如效果对比组件 `CssCompare`、画廊组件 `EffectGallery`）：

1. 在 `docs/.vitepress/theme/components/` 下创建组件
2. 在 `theme/index.ts` 中注册：

```typescript
import MyNewComponent from './components/MyNewComponent.vue'

export default {
  extends: DefaultTheme,
  enhanceApp({ app }) {
    app.component('MyNewComponent', MyNewComponent)
  },
}
```

3. 在任意 .md 文件中直接使用 `<MyNewComponent />`

## 12. 部署配置

### 12.1 Netlify 配置

```toml
[build]
  command = "npm run build"
  publish = "docs/.vitepress/dist"

[build.environment]
  NODE_VERSION = "20"

[[redirects]]
  from = "/*"
  to = "/404.html"
  status = 404
```

### 12.2 关键说明

| 配置项 | 说明 |
|--------|------|
| `publish` | 必须指向 VitePress 构建输出目录 |
| `NODE_VERSION` | 确保构建环境使用 Node 20 |
| 404 处理 | 所有未匹配路由重定向到 404 页面 |

### 12.3 构建产物

构建后生成纯静态文件：

```
docs/.vitepress/dist/
├── index.html
├── effects/
│   ├── index.html
│   ├── buttons/
│   │   ├── index.html
│   │   ├── glow-btn.html
│   │   └── ripple-btn.html
│   ├── backgrounds/
│   └── loaders/
├── guide/
│   └── how-to-add.html
├── assets/
│   ├── *.js       # Vue 组件 + 页面逻辑
│   └── *.css      # 样式文件
└── 404.html
```

所有产物均为静态 HTML + CSS + JS，无需服务器端运行时。

## 13. 安全设计

### 13.1 iframe 安全

| 威胁 | 防护措施 |
|------|----------|
| 效果代码访问宿主 DOM | sandbox 禁用 `allow-same-origin`，隔离文档上下文 |
| 效果代码窃取 Cookie | 禁用 `allow-same-origin`，无法访问宿主 Cookie |
| 效果代码修改页面 URL | 禁用 `allow-top-navigation` |
| 效果代码弹出广告 | 禁用 `allow-popups` |
| 恶意代码提交表单 | 禁用 `allow-forms` |

### 13.2 内容安全

- 所有效果代码由贡献者手动编写（非 UGC 平台）
- 代码存储在 Git 仓库中，有审计历史
- Markdown 内容中可嵌入的 HTML 被 Vue 模板编译器处理，XSS 风险低

## 14. 性能考虑

| 场景 | 性能特征 | 说明 |
|------|----------|------|
| 页面加载 | 静态 HTML 直接返回 | 无服务端渲染开销，CDN 缓存 |
| 效果预览 | 每个效果新增一个 iframe | 单页面展示效果数量少，影响可忽略 |
| 搜索 | 客户端全文搜索 | 索引文件在构建时生成，搜索瞬间完成 |
| 构建时间 | O(n) 随效果数量线性增长 | 100 个效果预计 < 30s |
